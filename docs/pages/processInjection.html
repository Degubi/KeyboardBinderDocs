Process Injection

Keyboard Binder has the functionality to inject code into processes. Some of the use-cases include:

  • Blocking normal keyboard inputs in applications when pressing keys that run scripts
  • Running some code when an application closes

Notes of Caution

In Keyboard Binder process injection is done manually.
The reasoning behind this is that some similar programs that e.g automatically block inputs causes most games to get the user banned because of the nature of process injection.
This of course implies that this kind of injection in games is a bad idea because of the high chance of getting banned!


Injecting

To inject into a process use the 'inject_into_process' function found in the 'KeyboardBinder' module, like so:

from keyboardBinder import KeyboardBinder

KeyboardBinder.inject_into_process('Adobe Premiere Pro.exe', True, lambda: print('PPro exited!'))

This code blocks the keyboard input in Premiere Pro and prints 'PPro exited!' when closing Premiere.
The keyboard which input gets blocked is the current profile's keyboard.

Note: you have to call this function once per session, so calling it on profile activation is a perfect place.
Calling 'inject_into_process' multiple times on the same process is a no-op.


Ejecting

Ejecting is done automatically when the profile gets disabled, or when Keyboard Binder closes.
